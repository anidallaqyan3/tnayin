Origin-ը հեռավոր շտեմարանի ստեղծված անունն է,որին ի սկզբանե կլոնավորել  է նախագիծը։ Այն օգտագործվում է պահեստի URL-ի փոխարեն և դրանով ավելի է հեշտացնում հղումը.
Git merge հրամանը միավորում է առանձին ճյուղեր և լուծում ցանկացած հակասական խմբագրում։ Երկու դեպքում միաձուլումը կարող է մտնել կոնֆլիկտային վիճակ:
Այն մի իրավիճակ է, որը տեղի է ունենում , երբ git-ը չի կարողանում ավտոմատ կերպով լուծել երկու պարտավորությունների միջև եղած տարբերությունը։
Git revert  հրամանը օգտագործվում է պահեստի պարտավորությունը, պատմությունը չեղարկելու համար։<<Հետարկել>> այլ հրամաններ ինչպիսիք են  git reset, git checkout HEARD։
Rebasing-ը նոր բազային հարցում տեղափոխելու կամ համակցելու գործընթաց է։Rebasing առավել օգտակար է և հեշտությամբ է պատկերացվում գործառույթների ճյուղավորման աշխատանքային հոսքի համատեքստում։Նրա հիմնական առավելությունը կայանում է նրանում , որ դուք ստանում ենք առավել մաքուր նախագիծ։ Նախ այն վերացնում է Git merge-ի կողմից պահանջվող անհարկի միաձուման պարտավորությունները։
